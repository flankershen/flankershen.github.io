SNMP配置及命令参数

1、SNMP命令包括：

2、snmp-server community

​ 2.1、设置团体名及访问权限：

​ 2.2、参数说明：
community-name为团体名，类型为字符串。
ro和rw表示该团体的访问权限有只读（read-only）和读写（read-write）两种。

​ 2.3、缺省情况
系统缺省认为具有只读权限的团体名为public，具有读写权限的团体名为private。
​ 2.4、命令模式
全局配置模式
​ 2.5、使用指南
SNMP v1 采用团体名认证方案，与设备认可的团体名不符的 SNMP 报文将被丢弃。不同的团体可具有只读（read-only）或读写（read-write）两种访问权限。具有只读权限的团体只能对设备信息进行查询，而具有读写权限的团体还可以对设备进行配置。
​ 2.6、举例
设置public团体具有只读权限，private团体具有读写权限

3、 snmp-server contact

设置网络管理人员的标识及联系方法

​ 3.1、参数说明
sysContact为一字符串，表示网络管理人员的标识和联系方法。
​ 3.2、缺省情况
系统缺省的sysContact为SysAdmin。
​ 3.3、命令模式
全局配置模式
​ 3.4、使用指南
当网络出现故障时，可以根据此信息与网络管理人员进行联系。
​ 3.5、举例

4、snmp-server enable traps

禁止或允许系统发送Trap。

​ 4.1、缺省情况
系统缺省为允许发送Trap。
​ 4.2、命令模式
全局配置模式
​ 4.3、使用指南
Trap是被管理设备主动向NMS发送的不经请求的信息，用于报告一些紧急的重要事件。
​ 4.4、举例
禁止系统发送Trap。

5、snmp-server host

设置Trap目标主机的IP地址。

​ 5.1、参数说明
host-ip-address为Trap目标主机的IP地址，为点分十进制格式。
​ 5.2、命令模式
全局配置模式
​ 5.3、使用指南
在允许系统发送Trap后，必须设置Trap目标主机的IP地址。
​ 5.4、举例
设置Trap目标主机IP地址为129.102.0.1。

​ 5.5、相关命令

###6、snmp-server location
设置路由器的物理位置描述。

​ 6.1、参数说明
sysLocation为路由器所在物理位置的描述。
​ 6.2、命令模式
全局配置模式
​ 6.3、使用指南
为了便于网络管理人员能够很快找到路由器，可以查看此关于路由器物理位置的描述。
​ 6.4、举例

###7、snmp-server trap-authentication snmpv1
允许或禁止路由器发送Authentication Trap。

​ 7.1、缺省情况
系统缺省设置为允许发送Authentication Trap。
​ 7.2、命令模式
全局配置模式
​ 7.3、使用指南
Authentication Trap 是 Trap 的一种。如果允许发送 Authentication Trap，则当被管设备收到含有错误团体名的请求时，可向 NMS 发送验证失败报告（Authentication Trap）。
​ 7.4、举例
禁止发送Authentication Trap。

ZTE交换路由设备配置的备份与恢复

一、TFTP服务器搭建

使用用户计算机搭建TFTP服务器，交换路由设备作为TFTP客户端。

  • 运行TFTPServer.exe，该程序所在的目录为TFTP的根目录。
  • 请保证您的TFTP可以Ping通所要执行后续任务的交换机的管理地址。

TFTPServer.exe所在的目录为：D:\Program Files\Cisco-TFTP-Server\Cisco TFTP Server 则TFTP的根目录为D:\Program Files\Cisco-TFTP-Server\Cisco TFTP Server。也可以更换根目录。
